# 8.5 · 避障与深度感知

球场上不止一个机器人。冲球、走位时撞到别人不仅影响发挥，还会被判犯规。本篇讲大脑怎么从**深度摄像头**"看见"障碍、建一张障碍地图，并供导航/追球/踢球查询绕行。

源码：`brain.cpp:2417`（建图）/ `2556`（查距离）/ `2581`（找安全方向）/ `2604`（算绕障方向）；碰撞时间 `msecsToCollide` 见 [8.2](./8.2-导航moveToPoseOnField.md)。

---

## 一、processDepthImage：深度图 → 障碍栅格（`brain.cpp:2417`）

深度相机每帧给一张"每像素到物体的距离"图。`processDepthImage` 把它处理成机器人周围的**占用栅格**。流程四步：

### 1) 反投影成机器人系三维点（`brain.cpp:2440`）

```cpp
for (int y = 0; y < height; y += sampleStep)          // 按 depth_sample_step 降采样
 for (int x = 0; x < width; x += sampleStep) {
    float depth = depthFloat.at<float>(y, x);
    if (depth > 0) {
        double x_cam = (x - cx) * depth / fx;          // 像素 + 深度 → 相机系三维点
        double y_cam = (y - cy) * depth / fy;
        Eigen::Vector4d point_cam(x_cam, y_cam, depth, 1.0);
        Eigen::Vector4d point_robot = data->camToRobot * point_cam;  // → 机器人系
        ...
```

用相机内参 `fx/fy/cx/cy`（由 `depthCameraInfoCallback` 从 `CameraInfo` 算得）把像素+深度反投影成相机系三维点，再乘 `camToRobot`（[8.3](./8.3-状态回调odom_imu_fall.md) 实时维护）转到机器人系。**降采样** `sampleStep` 是为了控制计算量——深度图几十万像素，没必要每个都处理。

### 2) 三道过滤（`brain.cpp:2460`）

只有同时满足的点才算"障碍"：

| 过滤 | 条件 | 滤掉什么 |
|------|------|----------|
| 高度阈值 | `point_robot.z > obstacle_min_height` | **地面**（太矮的点是草皮，不是障碍）|
| 排除自身 | 不在 `exclusion_x/y` 的身体框内 | 自己的身体/手 |
| 排除球 | 不在球附近 `ball_exclusion_radius/height` | **球本身**（球不是要避的障碍）|

> 💡 这三道滤波是建图的精髓。滤地面避免把整片草皮当障碍；滤自身避免把自己的躯干算进去；**滤球**尤其关键——机器人正要去踢球，要是把球当障碍躲开就本末倒置了。`ball.posToRobot` 来自视觉（[模块03](../03-视觉模块/index.md)），在这里被特意"挖空"。

### 3) 落进栅格统计占用（`brain.cpp:2489`）

```cpp
int grid_x = (point_robot(0) - x_min) / grid_size;
int grid_y = (point_robot(1) - y_min) / grid_size;
grid_occupied[grid_x][grid_y] += 1;                   // 该格内点数 +1（即 confidence）
```

机器人前方 `[0, max_x] × [-max_y, max_y]` 区域按 `grid_size` 划格，每个落入的有效点给对应格 +1。格里点越多、越可能是真障碍——这个计数就当作障碍 `confidence`。

### 4) 新旧障碍融合（`brain.cpp:2501`）

把本帧 `grid_occupied>0` 的格子转成 `GameObject`（标 `"Obstacle"`、记 `posToRobot`/`posToField`）。再处理旧障碍：**视野内**的旧障碍若本帧没再看到就清掉（眼见为实）、视野外的旧障碍保留（头转开了不代表障碍消失）、与新障碍太近的旧障碍丢弃防边界堆积。最后 `data->setObstacles(obs_new)`。超时清理在 `tick` 里另做。

## 二、distToObstacle：某方向最近障碍（`brain.cpp:2556`）

建好图后，避障决策只需问一个问题："沿某方向走，多远会撞上障碍？"

```cpp
double Brain::distToObstacle(double angle) {
    double minDist = 1e9;
    for (auto &o : data->getObstacles()) {
        if (o.confidence < occupancy_threshold) continue;       // 点太少→噪声，忽略
        Line line = {0, 0, cos(angle)*100, sin(angle)*100};     // 从自己出发的射线
        double perpDist = pointPerpDistToLine({o.x, o.y}, line);
        if (perpDist < collision_threshold) {                   // 障碍离射线够近→挡路
            double dist = 障碍在射线方向上的投影;                  // 内积
            if (dist > 0 && dist < minDist) minDist = dist;     // 取最近的
        }
    }
    return minDist;
}
```

> 💡 `occupancy_threshold` 过滤低置信障碍（点数太少多半是深度噪声）；`collision_threshold` 是"机器人半身宽"——障碍离行进射线的垂距小于它才算挡路。返回沿该方向最近障碍的距离，是所有避障逻辑的统一查询接口。

## 三、findSafeDirections / calcAvoidDir：往哪绕（`brain.cpp:2581` / `2604`）

```cpp
// findSafeDirections: 从 startAngle 起左右各扫，找第一个 distToObstacle > safeDist 的方向
vector<double> findSafeDirections(startAngle, safeDist, step) {
    for (angle = startAngle; angle < startAngle+PI; angle += step)  // 往左扫
        if (distToObstacle(angle) > safeDist) { safeAngleLeft = angle; leftFound=1; break; }
    for (angle = startAngle; angle > startAngle-PI; angle -= step)  // 往右扫
        if (distToObstacle(angle) > safeDist) { safeAngleRight = angle; rightFound=1; break; }
    return {leftFound, safeAngleLeft, rightFound, safeAngleRight};
}

// calcAvoidDir: 左右安全方向都有→选偏转角更小的那个（绕得最省）
double calcAvoidDir(startAngle, safeDist) {
    if (leftFound && rightFound) return fabs(angleLeft)<fabs(angleRight) ? angleLeft : angleRight;
    else if (leftFound) return angleLeft;
    else if (rightFound) return angleRight;
    else return 0;                                              // 全堵死→返回 0（直走/由调用方处理）
}
```

> 💡 `calcAvoidDir` 是导航避障（[8.2](./8.2-导航moveToPoseOnField.md) 的 v2/v3）的"决策核心"：从期望方向起，向左右各找第一个足够空的方向，挑**偏离最小**的那个绕——既避开障碍又不绕太远偏离目标。两边都堵就返回 0 交给上层（通常停下/后退重规划）。

## 四、谁在用避障？

避障不是默认全开，由各节点按配置选择性启用：

| 使用方 | 配置开关 / 参数 | 用途 |
|--------|----------------|------|
| `moveToPoseOnField/2/3` | `avoidObstacle` 入参、`safe_distance`、`avoid_secs` | 导航绕开障碍（[8.2](./8.2-导航moveToPoseOnField.md)）|
| `Chase` 追球 | `avoid_during_chase`、`oa_safe_dist` | 追球路上有障碍就 `calcAvoidDir` 绕（[7.5](../07-行为树与决策/7.5-动作节点-追球调整踢球.md)）|
| `Kick` 踢球 | `kick_ao_safe_dist`、`avoidPushing` | 踢前正前方有人→先后退 0.1 别撞（[7.5](../07-行为树与决策/7.5-动作节点-追球调整踢球.md)）|
| `cost` 协作 | 障碍惩罚项 | 路上有障碍的机器人 cost 加分、不优先去踢（[模块04](../04-裁判机与通信/index.md)）|

`cost` 计算（`updateCostToKick`，[模块04](../04-裁判机与通信/index.md)）里专门有一项"去球路上有没有障碍"的惩罚——挡路多的机器人即便离球近也未必该去踢，避免硬挤过人堆。

> 🏆 **避障对比赛是硬需求。** RoboCup 人形组明令禁止冲撞——撞到对手会被判 `HL_PHYSICAL_CONTACT`（身体接触犯规）甚至罚下（裁判机状态见 [模块04](../04-裁判机与通信/index.md)）。所以踢球前 `Kick` 宁可后退 0.1m 让一让也不硬踢，导航宁可绕远也不直撞。深度图障碍栅格（看静止障碍/近处遮挡）与 `msecsToCollide`（看场上机器人位置，[8.2](./8.2-导航moveToPoseOnField.md)）两套机制互补，前者来自视觉深度、后者来自队友/对手定位。

## 小结

- `processDepthImage`（`brain.cpp:2417`）把深度图**反投影成机器人系三维点 → 三道过滤（滤地面/自身/球）→ 落栅格计数 → 新旧融合**，得到障碍地图（点数即 confidence）。
- `distToObstacle(angle)`（`brain.cpp:2556`）是统一查询接口：沿某方向最近障碍的距离，用 `occupancy_threshold`/`collision_threshold` 滤噪声与判挡路。
- `findSafeDirections`（:2581）左右扫找空方向，`calcAvoidDir`（:2604）挑偏转最小的方向绕，两边堵死返回 0。
- 避障由 `moveToPoseOnField`/`Chase`/`Kick` 按 `avoid_during_chase`/`kick_ao_safe_dist` 等配置选择性启用；cost 里含障碍惩罚（[模块04](../04-裁判机与通信/index.md)）。
- 🏆 撞对手会被判 `HL_PHYSICAL_CONTACT`，避障是规则硬需求；深度栅格与 `msecsToCollide` 两套障碍来源互补。
