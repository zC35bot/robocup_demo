# 8.4 · 头部主动控制 HeadController

机器人的头（相机）能独立转动。怎么用好这颗"脖子上的眼睛"——盯球、丢了去找——是个独立的控制问题。Phase1 新增的 `HeadController`（`src/brain/src/head_controller.cpp`、`src/brain/include/head_controller.h`）是一个**可选的 ~50Hz 主动控头器**，把它从行为树的看球节点里独立出来。本篇逐函数讲它。

---

## 一、接管机制：`enable` 与 BT 让位

`HeadController::update` 在 `Brain::tick()` 末尾被调（`brain.cpp:444`），但有开关：

```cpp
// brain.cpp:444
if (config->headController.enable) {
    head_controller_.update(this);
}
```

开启后，行为树里原来的看球/扫场摄像头节点会**主动让位**，直接返回 SUCCESS 不下发头部指令（`brain_tree.cpp:152/231/260`）：

```cpp
// brain_tree.cpp:231  CamTrackBall 等节点开头
if (brain->config->headController.enable) return NodeStatus::SUCCESS; // Phase1 §5 takeover
```

> 💡 **为什么要短路 BT 节点？** 否则两边都给头下指令（BT 的看球节点 + HeadController），头会被来回拉扯抖动。所以约定：`enable=true` 时 HeadController 独占头部，BT 看球节点全部让位；默认 `enable=false`，行为不变（向后兼容）。这是典型的"新功能默认关、不影响已有比赛表现"的稳健做法。

## 二、`update`：50Hz 节流 + 模式分发（`head_controller.cpp:13`）

```cpp
constexpr double kUpdatePeriodSec = 0.02; // ~50Hz
void HeadController::update(Brain *brain) {
    double nowSec = now().seconds();
    if (haveLastSend_ && (nowSec - lastSendSec_) < kUpdatePeriodSec) return; // 节流到 50Hz

    Mode mode = selectMode(brain);                       // 选模式
    if (mode != lastMode_) {                             // 切模式时记下起始时刻
        if (mode == SEARCH_SWEEP)    sweepStartSec_    = nowSec;
        if (mode == SEARCH_DIRECTED) directedStartSec_ = nowSec;
    }
    lastMode_ = mode;

    switch (mode) {
        case TRACK:           trackBall(brain, pred100_robot); break;
        case SEARCH_DIRECTED: searchDirected(brain, brain->data->ballVel); break;
        case SEARCH_SWEEP:    default: searchSweep(brain); break;
    }
}
```

`Brain::tick()` 跑得比 50Hz 快，所以 `update` 自带节流——距上次发头指令不足 20ms 就直接返回。切换模式时记录该模式的"起始时刻"，让扫描相位连续（见三、四节）。`TRACK` 分支里如果定位可信（`using_field_frame`），把场地系预测 `pred100_field` 转到机器人系再追；否则直接用机器人系 `pred100_robot`。

## 三、`selectMode`：三模式选择（`head_controller.cpp:56`）

```cpp
Mode HeadController::selectMode(Brain *brain) const {
    if (brain->data->ball_prediction_valid) return Mode::TRACK;     // 有有效预测→盯着
    double sinceDetMs = brain->msecsSince(brain->data->ball.timePoint);
    double speed = norm(ballVel[0], ballVel[1]);
    if (sinceDetMs < 500.0 && speed > 0.1) return Mode::SEARCH_DIRECTED; // 刚丢且球在动→定向找
    return Mode::SEARCH_SWEEP;                                       // 否则→地毯式扫
}
```

| 模式 | 触发条件 | 策略 |
|------|----------|------|
| `TRACK` | 球预测有效（`ball_prediction_valid`）| 把头对准球的预测位 `pred100` |
| `SEARCH_DIRECTED` | 丢球 < 500ms 且球速 > 0.1 | 沿球速度方向小幅扫，快速重捕 |
| `SEARCH_SWEEP` | 球丢久了/静止 | 左右三角扫 + 上下俯仰交替，地毯式找 |

## 四、三个动作函数

### 4.1 trackBall：对准预测位（`head_controller.cpp:68`）

```cpp
void HeadController::trackBall(Brain *brain, const double pred100_robot[2]) {
    double x = pred100_robot[0], y = pred100_robot[1];
    double range = max(norm(x, y), 1e-3);
    double yaw   = atan2(y, x);                                  // 头转向球方位
    double pitch = atan2(brain->config->get_robot_height(), range); // 越近越低头
    sendHead(brain, pitch, yaw);
}
```

> 💡 追的是 **pred100**（球 100ms 后的预测位，[模块06](../06-定位与球预测/index.md)）而非当前球位——头转动也有延迟，打提前量才能稳稳盯住运动的球。`pitch` 由"机器人身高 / 到球水平距离"反正切给出：球越近、俯角越大（越低头看脚下），几何上正对球心。

### 4.2 searchDirected：沿球速方向扫（`head_controller.cpp:80`）

```cpp
double dirField  = atan2(vel_field[1], vel_field[0]);
double centerYaw = toPInPI(dirField - robotPoseToField.theta);  // 球速方向（机器人系）为中心
double yaw = centerYaw + deg2rad(30) * sin(2*M_PI*0.7*t);       // 以 0.7Hz、±30° 摆动
double pitch = 0.4;
sendHead(brain, pitch, yaw);
```

刚丢球且球在跑时，球大概率往它原来的速度方向去了。于是以**球速方向**为中心做 ±30°、0.7Hz 的正弦小摆扫描，快速重新捕获——比从头乱扫高效得多。`t` 从进入该模式时刻起算（`directedStartSec_`），保证相位连续不跳变。

### 4.3 searchSweep：三角扫 + 俯仰交替（`head_controller.cpp:95`）

```cpp
double phase = fmod(t, 2.0) / 2.0;                              // 2 秒一周期(0.5Hz)，相位[0,1)
double tri = phase < 0.5 ? phase*2.0 : 2.0 - phase*2.0;         // 三角波 0→1→0
double yaw = rightYaw + (leftYaw - rightYaw) * tri;            // 在左右限位间线性扫
double pitch = phase < 0.5 ? 0.5 : 0.2;                         // 前半周期低头、后半抬头
sendHead(brain, pitch, yaw);
```

彻底丢球时，在 yaw 左右限位之间用**三角波**匀速来回扫（不像正弦在两端慢、中间快），同时半周期低头(0.5)、半周期抬头(0.2)交替——把"近处地面"和"远处"都覆盖到，地毯式找球。

### 4.4 sendHead：角度钳制（`head_controller.cpp:113`）

```cpp
void HeadController::sendHead(Brain *brain, double pitch, double yaw) {
    yaw   = cap(yaw, leftYaw, rightYaw);    // yaw 钳进左右限位（配置约 ±60°）
    pitch = cap(pitch, 0.7, 0.0);           // pitch 钳进安全俯仰带 [0, 0.7]
    brain->client->moveHead(pitch, yaw);    // 真正下发（moveHead 内还会再钳一次）
    haveLastSend_ = true; lastSendSec_ = now().seconds();
}
```

所有模式算出的角度都过 `sendHead` 钳一遍：`yaw` 限在左右限位（配置里约 ±60°）、`pitch` 限在 `[0, 0.7]`。下游 `moveHead`（[8.1](./8.1-RobotClient指令翻译.md)）还会再钳一次——**双重保险**，绝不让脖子转到机械极限外。

## 五、为什么要主动控头？

> 💡 **脖子比身体转得快得多，也稳得多。** 与其让整个机器人转身去找球（慢、还可能把球带歪、容易失稳），不如先让头去追、去扫——既快又不影响双足走位，还能持续把球喂给预测器（[模块06](../06-定位与球预测/index.md)）保持预测有效。把控头从行为树里独立出来、用专门的 50Hz 高频循环驱动，也比"每个 BT tick 顺手发一次头指令"更平滑（BT tick 频率不稳）。这是"感知-控制分层"的又一例：身体管走位、头管视野，各司其职。

---

## 小结

- `HeadController`（`head_controller.cpp`）是可选的 ~50Hz 主动控头器，`config.headController.enable` 开启时接管头部，BT 看球节点让位（`brain_tree.cpp:152/231/260`）。
- `update`（:13）自带 50Hz 节流并按 `selectMode`（:56）分发三模式：`TRACK`（有预测→盯球）/ `SEARCH_DIRECTED`（刚丢且球动→沿球速扫）/ `SEARCH_SWEEP`（丢久→三角扫）。
- `trackBall`（:68）追 **pred100** 并按距离算俯角；`searchDirected`（:80）以球速方向为中心 ±30° 正弦摆；`searchSweep`（:95）左右三角扫 + 俯仰交替。
- `sendHead`（:113）把 yaw 钳进 ±60°、pitch 钳进 [0,0.7]，与 `moveHead` 双重保险。
- 设计动机：脖子比身体转得快且稳，用头追/扫球不影响走位，还能持续喂预测器（[模块06](../06-定位与球预测/index.md)）。
