# 7.5 · 动作节点 · 追球/调整/踢球

前面几篇讲"决策"——大脑把结论写成 `decision` 字符串。本篇讲**最核心的四个动作节点**，它们把决策翻译成真正下发给底层的速度指令 `(vx, vy, vθ)`：`Chase`（追球）、`Adjust`（绕球调整）、`Kick`（蟹步踢球）、`RLVisionKick`（视觉踢球）。

源码：`brain_tree.cpp:280/682/1110/1264`。底层运动接口（`client->setVelocity`/`crabWalk`/`RLVisionKick`）见 [模块08](../08-机器人控制与底层/index.md)。

> 关键提醒（[7.1](./7.1-行为树框架与黑板.md) 已铺垫）：`Chase`/`Adjust` 是 **`SyncActionNode`**——每帧瞬间算一次速度就返回 `SUCCESS`，靠外层 `ReactiveSequence` 每帧重进来。`Kick`/`RLVisionKick` 是 **`StatefulActionNode`**——踢球是有头有尾的过程，用 `onStart/onRunning/onHalted` 跨多帧执行。

## 一、Chase：追球（`brain_tree.cpp:280`）

`Chase` 不是直愣愣冲向球，而是**冲向"球后方一个能起脚的点"**，并且追的是**预测球位**而非当前球位。

### 1.0 进入即防呆：球位未知/出界就停（`brain_tree.cpp:287`）

`Chase::tick` 一进来先做一道**安全闸**，不满足前提直接站定退出，绝不带着无效球位往下算：

```cpp
if (
    !brain->tree->getEntry<bool>("ball_location_known")   // 根本不知道球在哪
    || brain->isBallOut(3.0, 1.5)                          // 或球已判出界（阈值3.0、可信球距1.5）
) {
    brain->client->setVelocity(0, 0, 0);
    return NodeStatus::SUCCESS;
}
```

> 💡 **安全加固：`Chase` 新增 ballOut 兜底。** 追球逻辑后面要用 `ball.posToField`、`kickDir` 等算"绕后/直冲"的目标点——如果球位根本不可信（没定位到球）或球已经出界，这些计算就是在**追一个不该追的球**：轻则朝界外冲、把自己带出场，重则用陈旧/越界坐标算出乱七八糟的绕行切线。这里在函数最前面直接判 `ball_location_known` 和 `isBallOut(3.0, 1.5)`，命中就 `setVelocity(0,0,0)` 站定并返回 `SUCCESS`，把"要不要继续追"交回外层 `ReactiveSequence` 下一帧重判（此时多半会切到 `GoBackInField` 或 `find`）。注意这里用的 `isBallOut(threshold=3.0, maxRange=1.5)` 比黑板里的 `ball_out` 更严——只有球确实在近处（`range<1.5`）且越界才停，避免远处噪声误停。

### 1.1 追预测位 pred100（`brain_tree.cpp:304`）

```cpp
Point2D predBallField;
bool usePred = brain->data->ball_prediction_valid;
if (usePred && using_field_frame)        predBallField = pred100_field;       // 场地系预测
else if (usePred)                        predBallField = robot2field(pred100_robot); // 机器人系预测转场地系
else                                     predBallField = ball.posToField;      // 没预测→用当前球位
```

> 💡 为什么追预测位？因为"看到球→算决策→下发速度→机器人真动起来"有几十毫秒延迟。如果追当前球位，等机器人到了，球早跑了。`pred100` 是球 **100ms 后**的预测位置（[模块06](../06-定位与球预测/index.md)），追它相当于"打提前量"，补偿感知+控制延迟。定位可信时用场地系预测，否则用机器人系预测兜底，再没有就退回当前球位。

### 1.2 direct vs circle_back：要不要绕后（`brain_tree.cpp:347`）

算出目标点 `target_f` 时分两种情况——关键看"机器人→球"的方向 `theta_rb` 和"想踢的方向" `kickDir` 夹角：

```cpp
static string targetType = "direct";
double dirThreshold = M_PI/2;
if (targetType == "direct") dirThreshold *= 1.2;     // 滞回：已 direct 就更难切走

if (fabs(toPInPI(kickDir - theta_rb)) < dirThreshold) {
    targetType = "direct";                            // 我在球的"踢球方向后侧"
    target_f.x = ballPos.x - dist*cos(kickDir);       // 直接走到球后 dist 处
    target_f.y = ballPos.y - dist*sin(kickDir);
} else {
    targetType = "circle_back";                       // 我在球的错误一侧，要绕
    circleBackDir = toPInPI(theta_br - kickDir) > cbDirThreshold ? 1.0 : -1.0;  // 选绕行方向
    double tanTheta = theta_br + circleBackDir * acos(clamp(safeDist/max(ballRange, 1e-5), -1.0, 1.0));
    target_f.x = ballPos.x + safeDist*cos(tanTheta);  // 沿安全圈切线绕到球后
    target_f.y = ballPos.y + safeDist*sin(tanTheta);
}
```

> 💡 **为什么要绕后（circle_back）？** 假设球在我和对方球门之间，但我站在球的"球门那一侧"——如果直接冲过去，会把球往**自己门里**捅。所以要先**绕到球的后方**（远离己方门那侧），让"我→球→对方门"成一条线，再往前推。`circle_back` 沿着以球为心、`safeDist` 为半径的圆**切线**绕行，既靠近球又不碰到球（碰球会犯规或乱带）。`direct` 则是已经站对了一侧，直接走到球后 `dist` 处准备起脚。`dirThreshold *= 1.2` 的滞回避免在两种模式边界反复切换。

> 💡 **安全加固：`acos` 输入 clamp 到 [-1, 1]。** 切线角 `tanTheta` 里的 `acos(safeDist/ballRange)` 在**球比安全半径还近**时（`ballRange < safeDist`）参数会 `>1`，此时 `acos` 返回 **NaN**，`tanTheta`→NaN→目标点 NaN→下发 NaN 速度，机器人步态会瞬间失控。旧代码只用 `min(1.0, …)` 卡了上界，但没防**分母为 0**（`ballRange==0` 时除零得 `inf`）和下界。新写法 `acos(clamp(safeDist/max(ballRange, 1e-5), -1.0, 1.0))` 一次堵死两个坑：`max(ballRange, 1e-5)` 防除零，`clamp(…, -1.0, 1.0)` 把商夹进 `acos` 的合法定义域，彻底杜绝 NaN。同一提交也给几何工具 `angleBetweenLines`（`utils/math.h:160`）的 `acos` 加了**同样的** `clamp(…, -1.0, 1.0)`，属于同一类"浮点定义域防呆"。

### 1.3 避障与速度（`brain_tree.cpp:364`）

```cpp
double targetDir = atan2(target_r.y, target_r.x);
if (avoidObstacle && distToObstacle(targetDir) < oaSafeDist) {    // 路上有障碍
    auto avoidDir = brain->calcAvoidDir(targetDir, oaSafeDist);    // 算绕障方向
    vx = 0.5*cos(avoidDir); vy = 0.5*sin(avoidDir); vtheta = ballYaw;
} else {
    vx = min(vxLimit, ball.range);                                 // 越近越慢
    vy = 0; vtheta = targetDir;
    if (fabs(targetDir)<0.1 && ballRange>2.0) vtheta = 0.0;        // 远且基本朝向→不转
    vx *= sigmoid(fabs(vtheta), 1, 3);                            // 要大转弯→先减速
}
```

- **正前推进 + 转向对准**：`vx` 往前、`vtheta` 转向目标点。`vx *= sigmoid(|vtheta|,...)`——需要大角度转向时压低前进速度，先转再走，避免画大弧。
- 还有**近球限速**（`brain_tree.cpp:298`）：球进到 `near_ball_range` 内时 `vxLimit` 压到 `near_ball_speed_limit`，防止高速撞球。

### 1.4 速度平滑（`brain_tree.cpp:385`）

```cpp
static double smoothVx=0, smoothVy=0, smoothVtheta=0;
smoothVx = smoothVx*0.7 + vx*0.3;     // 指数滤波
...
brain->client->setVelocity(vx, vy, vtheta);
```

用 0.7/0.3 的一阶指数滤波平滑速度，避免每帧速度突变让步态发抖（注意代码里平滑量算了但下发的是原始 vx/vy/vtheta，平滑主要起记忆/可视化参考）。

### 1.5 SimpleChase：手柄辅助的朴素追球（`brain_tree.cpp:405`）

`SimpleChase` 是 `Chase` 的"简化版"，用在手柄辅助模式（`assist_chase`，见 [7.2](./7.2-主树game_xml.md)）——不算绕后、不追预测位，直接把"机器人系下的球位"当速度，乘一个随偏航角衰减的 `linearFactor` 后限幅下发：

```cpp
double vx = ball.posToRobot.x;
double vy = ball.posToRobot.y;
double vtheta = ball.yawToRobot * 4.0;
double linearFactor = 1/(1+exp(3*(ball.range*fabs(ball.yawToRobot)) - 3));  // 球越偏越慢
vx *= linearFactor; vy *= linearFactor;

vx = cap(vx, vxLimit, -0.25);      // 前后：正向到 vxLimit，后退最多 -0.25
vy = cap(vy, vyLimit, -vyLimit);
vtheta = cap(vtheta, 2.0, -2.0);   // 转向：夹到 ±2.0 rad/s
```

> 💡 **安全加固：给后退速度和转向速度封顶。** 改动有两处：① `vx` 的下界从 `-1.0` 收紧到 **`-0.25`**——`vx` 直接来自 `ball.posToRobot.x`，当球在身后（负 x）时 `vx` 会是较大负值，旧的 `-1.0` 允许机器人**高速倒退**，双足倒走本就极易失稳、还看不见身后障碍；封到 `-0.25` 让它最多慢速微退，逼它靠转身（`vtheta`）而不是猛倒车去够身后的球。② 新增 `vtheta = cap(vtheta, 2.0, -2.0)`——`vtheta = yawToRobot*4.0` 在球大角度偏离时会算出很大的角速度，不封顶会让机器人**原地急旋**、步态发散甚至摔倒；夹到 `±2.0 rad/s` 保证转向平稳可控。这与下面 `Assist`（[7.6](./7.6-找球与移动节点.md) §3.3）拿到的是**同一组**封顶（`-0.25` / `±2.0`），是这次提交对所有"直接拿球位当速度"节点统一加的稳定性护栏。

## 二、Adjust：绕球微调对准（`brain_tree.cpp:682`）

球已经到脚边但**没对准踢球方向**时（`decision=="adjust"`），`Adjust` 让机器人**贴着球绕半圈**，把身体转到正对 `kickDir`，同时保持和球的距离。这是踢球前的"最后对准"。

### 2.1 用 pred300（`brain_tree.cpp:712`）

```cpp
bool usePred300 = getEntry<bool>("pred300_valid") && using_field_frame;
if (usePred300) {                                  // 用 300ms 预测位
    refRobot = field2robot(pred300_field);
    ballRange = norm(refRobot.x, refRobot.y);
    ballYaw   = atan2(refRobot.y, refRobot.x);
    dir_rb_f  = atan2(pred300_field.y - robot.y, pred300_field.x - robot.x);
} else {                                           // 兜底：当前球位
    dir_rb_f = robotBallAngleToField;
    ballRange = ball.range; ballYaw = ball.yawToRobot;
}
```

> 💡 `Adjust` 用 **300ms** 预测（比 `Chase` 的 100ms 更长），因为绕球调整是个慢动作、要绕半圈，提前量要打得更足。[模块06](../06-定位与球预测/index.md) 提供这些预测。

### 2.2 切向 + 径向速度分解（`brain_tree.cpp:730`）

这是 `Adjust` 的数学核心。把目标速度拆成两个分量：

```cpp
double deltaDir = toPInPI(kickDir - dir_rb_f);     // 当前对准还差多少角
double st = st_far;                                // 切向速度（绕球）
double sr = cap(R - r, 0.5, 0);                    // 径向速度（拉到目标半径 r）
if (fabs(deltaDir)*R < NEAR_THRESHOLD) st = st_near; // 快对准了→切向减速精调

// 切向方向：垂直于"机器人→球"，朝缩小 deltaDir 的一侧
double thetat_r = dir_rb_f + M_PI/2 * (deltaDir>0 ? -1 : 1) - theta_robot_f;
double thetar_r = dir_rb_f - theta_robot_f;        // 径向方向：朝球

vx = st*cos(thetat_r) + sr*cos(thetar_r);          // 两分量合成
vy = st*sin(thetat_r) + sr*sin(thetar_r);
vtheta = ballYaw * vtheta_factor;                  // 同时转身正对球
```

> 💡 **切向 st** 让机器人沿着以球为圆心的圆**绕行**（改变"机器人在球的哪个方位角"，从而对准 kickDir）；**径向 sr** 把机器人和球的距离拉到目标半径 `r`（`sr = cap(R-r,...)`，离得远就往里收）。两者合成就是"边绕边保持距离"。快对准时（`|deltaDir|*R < NEAR_THRESHOLD`）切向从 `st_far` 切到更小的 `st_near` 做精细对准——粗调快、精调慢，和决策里的滞回是一个思路。

### 2.3 vtheta 对准 + TURN_FIRST（`brain_tree.cpp:751`）

```cpp
if (fabs(ballYaw) < NO_TURN_THRESHOLD) vtheta = 0.;          // 已基本正对球→不转
if (fabs(ballYaw) > TURN_FIRST_THRESHOLD && fabs(deltaDir) < M_PI/4) {
    vx = 0; vy = 0;                                          // 球偏太多→先原地转正对球，别平移
}
```

> 💡 `TURN_FIRST`：当球严重偏离正前方（`|ballYaw|` 大）但踢球方向已经差不多对了，就**先把身体转到正对球**（vx/vy 清零，只留 vtheta），转好了再绕。否则一边大角度平移一边转，容易踩到球或失稳。这是"先转后移"的稳健策略。

## 三、Kick：蟹步踢球（`brain_tree.cpp:1110/1150`）

`Kick` 是 `StatefulActionNode`。它用 **crabWalk（蟹步）**——机器人侧向迈步把球"铲"出去，分**启动**和**持续加速**两阶段，并有**四维中止**保护。

### 3.1 _calcSpeed：踢球速度几何（`brain_tree.cpp:1110`）

```cpp
double adjustedYaw = ball.yawToRobot + yawOffset;            // 加偏置补偿
double tx = cos(adjustedYaw)*ball.range;                     // 球相对机器人的 x
double ty = sin(adjustedYaw)*ball.range;                     // 球相对机器人的 y
if (fabs(ty)<0.01 && fabs(adjustedYaw)<0.01) { vx=vxLimit; vy=0; }   // 正前方→直推
else {
    vy = ty>0 ? vyLimit : -vyLimit;                          // 侧向先给满
    vx = vy/ty*tx*vxFactor;                                  // 按球方位配前进速度
    if (fabs(vx)>vxLimit) { vy *= vxLimit/vx; vx = vxLimit; } // 限幅
}
```

让速度矢量大致指向球，把球往正前蹭出去。

### 3.2 onStart：启动 + 踢前避障（`brain_tree.cpp:1150`）

```cpp
_minRange = ball.range; _speed = 0.5; _startTime = now();
if (avoidPushing && role!="goal_keeper" && 不在球门区附近 && distToObstacle(ballYaw) < kickAoSafeDist) {
    brain->client->setVelocity(-0.1, 0, 0);                  // 前方有障碍→先后退 0.1，别撞
    return SUCCESS;
}
double angle = ball.yawToRobot;
brain->client->crabWalk(angle, _speed);                      // 启动蟹步，初速 0.5
return RUNNING;
```

> 🏆 踢球前若正前方有障碍（多半是对方球员），先后退 0.1m 避让——RoboCup 禁止推人犯规，硬踢可能撞上对手。守门员不避（清球优先）。

### 3.3 onRunning：逐步加速 + 四维中止（`brain_tree.cpp:1176`）

每帧把蟹步速度往上加 `_speed += 0.1`（`brain_tree.cpp:1247`），逐渐发力踢球；同时检查**四种中止条件**，任一触发就放弃这次踢球、退回追球：

| # | 中止条件 | `setKickAbort` 编码 | 含义 |
|---|----------|---------------------|------|
| 1 | 球远离了 `ball.range - _minRange > 阈值` 或球丢 1 秒 | `setKickAbort(1)` | ball_moved：球被抢走/踢飞了 |
| 2 | `ball_confidence < abort_confidence` | `setKickAbort(2)` | confidence：看不清球了 |
| 3 | `recoveryState == HAS_FALLEN` | `setKickAbort(3)` | fall：摔倒了 |
| 4 | `fabs(ball.yawToRobot) > abort_yaw` | `setKickAbort(4)` | yaw：球偏出可踢角度了 |

```cpp
if (ballRange - _minRange > MOVE_RANGE_THRESHOLD || 球丢) { setKickAbort(1); return SUCCESS; }
if (recoveryState==HAS_FALLEN)          { setKickAbort(3); return SUCCESS; }
if (ball_confidence < abort_confidence) { setKickAbort(2); return SUCCESS; }
if (fabs(ball.yawToRobot) > abortYaw)   { setKickAbort(4); return SUCCESS; }
...
if (msecsSince(_startTime) > msecs) { setVelocity(0,0,0); return SUCCESS; }   // 踢够时间→结束
if (ballDetected) { _speed += 0.1; crabWalk(angle, min(speedLimit, _speed)); } // 继续加速踢
```

> 💡 **四维中止**是踢球的安全网：踢球是个不可逆的发力动作，一旦球况变了（被抢/看不清/摔了/球偏了），死踢空气毫无意义还浪费时间。每种情况用不同的 `setKickAbort` 编码（1/2/3/4）记录原因，便于回放分析"为什么这一脚没踢成"。`_minRange` 记录踢球过程中球的最近距离，球比它远出一截就判定"球已离开脚"。中止时返回 `SUCCESS`（动作正常结束），外层 `ReactiveSequence` 下帧重新决策，`KickSelector` 的分支 1/2 会把决策降级回 chase（[7.3](./7.3-前锋决策.md)）。

### 3.4 onHalted：被打断时复位（`brain_tree.cpp:1255`）

```cpp
void Kick::onHalted() { _startTime -= rclcpp::Duration(100, 0); }
```

> 💡 `Kick` 被更高优先级决策打断时（比如球突然出界），框架调 `onHalted`。这里把 `_startTime` 往前拨 100 秒，效果是"下次进来 `msecsSince(_startTime)` 立刻超时"，让残留的踢球状态作废、干净退出，不会带着旧计时进入下一次踢球。这正是 [7.1](./7.1-行为树框架与黑板.md) 说的"Stateful 节点必须在 onHalted 收尾"。

## 四、RLVisionKick：视觉踢球（`brain_tree.cpp:1264`）

`auto_visual_kick` 决策走这里。它**不自己算踢球速度**，而是减速站稳后把控制权交给底层的 `client->RLVisionKick()`（一个训练好的视觉踢球策略，见 [模块08](../08-机器人控制与底层/index.md)），自己只负责"减速→启动→低头看球→满足条件就退出"。

### 4.1 流程（`brain_tree.cpp:1264-1334`）

```cpp
onStart():   startDecelerate(500); stepDecelerate(); return RUNNING;  // 先花 500ms 减速

onRunning():
  if (shouldExitRLVisionKick) { 清标志; recordExitTime(); return SUCCESS; }  // 外部要求退出
  if (_isDecelerating) {
     stepDecelerate();                          // 持续减速到 0
     if (减速完成) {
        if (_pendingRobocupWalk) { robocupWalk(); return SUCCESS; }  // 退出收尾
        else if (!_visionKickStarted) { client->RLVisionKick(); _visionKickStarted=true; }  // 启动视觉踢
     }
     return RUNNING;
  }
  if (_visionKickStarted) {                      // 配合低头看球
     if (headMsec<300) moveHead(0.4, 0.0);
     else if (headMsec<550) moveHead(0.7, 0.0);
  }
  // 退出条件（brain_tree.cpp:1340）
  double maxMsecKick = 15000.0;                       // 硬性时限：视觉踢最多持续 15 秒
  bool ballTooFar = ballDetected && ball.range > 1.5; // 球距门槛从 5.0 收紧到 1.5
  bool shouldExit = ((ballTooFar || tmMyCost>8.0) && elapsed>minMsecKick)
                    || elapsed > maxMsecKick          // 超 15 秒无条件退出
                    || lose_ball || ball_out;
  if (shouldExit) { recordExitTime(); startDecelerate(500); _pendingRobocupWalk=true; ...; return RUNNING; }
  return RUNNING;
```

- **先减速 500ms**：交给底层策略前要先稳住身体（视觉踢策略需要一个干净的初始姿态）。
- **低头看球**：`moveHead(0.4→0.7, 0)` 让相机持续盯着脚下的球，配合视觉踢策略。
- **退出条件**：球太远（**>1.5m**）、cost 太高（>8）、**已踢超 15 秒（`maxMsecKick`）**、丢球（`lose_ball`）、球出界（`ball_out`）任一满足就退出，退出时再减速并切回普通 `robocupWalk` 步态。

> 💡 **安全加固：收紧 `RLVisionKick` 的退出条件。** 两处改动都在"该早点收手了"这个方向上加码：① **球距门槛 `5.0m → 1.5m`**——视觉踢是"到球边、把球踢出去"的近距离动作，只有球真在脚下（`<1.5m`）才有意义；旧的 `5.0m` 太宽松，会让机器人在球都跑出去老远、早该改成 `Chase` 追球时还**赖在视觉踢里空踢**，白白拖延。② **新增 15 秒硬性时限 `maxMsecKick`**——原来的退出全靠"球远/cost 高/丢球/出界"这些外部条件，万一它们都不触发（比如球一直卡在脚边但踢不动），机器人可能**无限期滞留**在视觉踢状态。加一条 `elapsed > 15000` 的兜底，保证无论如何最多踢 15 秒就退出、切回正常步态重新决策，避免卡死在一个不再有效的动作里。

### 4.2 静态防抖 _lastExitTime（`brain_tree.cpp:1262`）

```cpp
static rclcpp::Time RLVisionKick::_lastExitTime;   // 类静态成员
void recordExitTime() { _lastExitTime = now(); }
```

> 💡 `_lastExitTime` 是**静态**的，每次退出视觉踢都记录时间，用于"刚踢完一次别马上又触发"的防抖（视觉踢是个重动作，频繁进出会让机器人来回切步态而不稳）。`onHalted`（`brain_tree.cpp:1336`）被打断时也会 `setVelocity(0)` + `robocupWalk()` + `recordExitTime()` 干净收尾，并清掉 `tmImInVisualKick`。

## 小结

- `Chase`（`brain_tree.cpp:280`）追 **pred100** 预测位补偿延迟，用 direct/circle_back 决定是否绕到球后（避免把球捅进自家门），带避障、近球限速、速度平滑；进入即判 `ball_location_known`/`isBallOut(3.0,1.5)` 防呆，circle_back 的 `acos` 输入 `clamp` 到 [-1,1]（配 `max(ballRange,1e-5)` 防除零）杜绝 NaN。
- `SimpleChase`（`brain_tree.cpp:405`）是手柄辅助的朴素追球，直接拿球位当速度；后退速度封到 `-0.25`、`vtheta` 封到 `±2.0`，防高速倒退/急旋失稳（与 `Assist` 同一组封顶）。
- `Adjust`（`brain_tree.cpp:682`）用 **pred300**，把速度分解成**切向（绕球对准）+ 径向（保持距离）**，快对准时切到 near 速度精调，球偏太多时 TURN_FIRST 先转后移。
- `Kick`（`brain_tree.cpp:1150`）蟹步两阶段（启动 + `_speed+=0.1` 逐步加速），**四维中止**（ball_moved/confidence/fall/yaw，各自 `setKickAbort(1~4)`），踢前避障后退 0.1，`onHalted` 拨快计时复位。
- `RLVisionKick`（`brain_tree.cpp:1264`）减速 500ms 后交给底层 `client->RLVisionKick()` 并低头看球，退出条件已收紧——球距门槛 `5.0→1.5m`、新增 15 秒硬性时限（`maxMsecKick`），再叠加 cost 高/丢球/出界，避免空踢滞留；`_lastExitTime` 静态防抖。
- `Chase`/`Adjust` 是 Sync（每帧重算），`Kick`/`RLVisionKick` 是 Stateful（跨帧、有 onHalted 收尾）。
