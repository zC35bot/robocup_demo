# 6.1 · 自定位 Locator（粒子滤波）

机器人没有 GPS，要靠**看到的场地特征**反推自己在场上的绝对位置。`Locator`（`src/brain/src/locator.cpp` / `locator.h`）用**粒子滤波 + 收缩搜索**解这个问题。本篇逐函数讲透算法。

---

## 一、问题与思路

**问题**：已知场地的真值地图（角点/线在哪），已知机器人此刻看到的角点（在机器人系），求机器人在场地的位姿 (x,y,θ)。

**思路**：把位姿当未知数，撒一堆候选"粒子"去猜。对每个候选位姿，把"机器人看到的角点"用它变换到场地系，看和真值地图对得齐不齐——对得越齐，这个候选越可能是真的。逐轮淘汰差的、在好的附近细化，直到收敛。

> 💡 **为什么用粒子滤波而不是直接解方程？** 角点匹配存在**歧义**：四个 L 角点长得一模一样，对称场地两端镜像对称，单个角点能对应多个真值。这是个**多峰**问题（多个局部最优）。粒子滤波天然能并行探索多个假设，靠"哪个假设让**所有**观测都自洽"来胜出，比梯度下降这种容易卡在局部最优的方法鲁棒得多。

数据结构：`hypos` 是一个 `num×6` 的 Eigen 矩阵，每行一个粒子：`[x, y, theta, 残差, 概率, 累积概率]`。

## 二、真值角点地图 `calcFieldMarkers`（`locator.cpp:34`）

初始化时（`init` 调用）按场地尺寸算出所有真实角点：
```cpp
// X 型：中圈与中线的两个交点
push(X, 0, -circleRadius);  push(X, 0, circleRadius);
// P 型：两个罚球点
push(P, length/2-penaltyDist, 0);  push(P, -length/2+penaltyDist, 0);
// T 型：中线两端边线 + 罚球区/球门区与底线的交点
push(T, 0, width/2);  push(T, 0, -width/2);
push(T, length/2, penaltyAreaWidth/2);  ... // 罚球区
push(T, length/2, goalAreaWidth/2);  ...     // 球门区
// L 型：罚球区/球门区的角 + 场地四角
push(L, length/2-penaltyAreaLength, penaltyAreaWidth/2);  ...
push(L, length/2, width/2);  ...              // 四角
```
> 这些坐标都在**场地系**（中心为原点，见 [模块05 · 5.4](../05-大脑数据与坐标系/5.4-三套坐标系与变换.md)）。一共几十个角点，分 X/P/T/L 四类。匹配时**只在同类型间**算距离——这是减少歧义的关键。

## 三、坐标变换 `markerToFieldFrame`（`locator.cpp:70`）

把机器人系下的一个角点，按某个候选位姿变换到场地系：
```cpp
FieldMarker markerToFieldFrame(marker_r, pose_r2f) {
    // 3×3 齐次变换矩阵 T = [cosθ -sinθ x; sinθ cosθ y; 0 0 1]
    Eigen::Matrix3d transform;
    transform << cos(theta), -sin(theta), x,
                 sin(theta),  cos(theta), y,
                 0, 0, 1;
    point_f = transform * [marker_r.x, marker_r.y, 1];
    return {marker_r.type, point_f.x, point_f.y, ...};
}
```
即"如果机器人真在 `pose` 处，那它看到的这个角点应该在场地的哪"。

## 四、残差：候选位姿有多好 `residual`（`locator.cpp:182`）

```cpp
double residual(markers_r, pose) {
    double res = 0;
    for (auto marker_r : markers_r) {
        double dist = max(norm(marker_r.x, marker_r.y), 0.1);   // 角点离机器人的距离
        auto marker_f = markerToFieldFrame(marker_r, pose);     // 变换到场地系
        res += minDist(marker_f) / dist * 3;                    // 到最近同类真值角点的距离/dist
    }
    return res;
}
```
- `minDist(marker)`（`:141`）：该角点到**同类型**最近真值角点的距离。
- **除以 `dist`（角点离机器人的距离）做加权**：近处看到的角点更可信（视觉远处误差大），所以近角点的偏差权重更高。
> 💡 残差衡量"这个候选位姿下，所有观测角点和真值地图的总偏差"。残差越小，候选越可能是真位姿。

## 五、概率与重采样

### `calcProbs`（`locator.cpp:219`）—— 把残差变概率
```cpp
for (每个粒子) {
    res = residual(markers_r, pose);
    hypos(i,3) = res;
    if (res < bestResidual) { bestResidual = res; bestPose = pose; }  // 记最优
}
// 用高斯把残差转概率：残差小→概率高
double sigma = 残差的标准差;
double mu = 最小残差 - muOffset * sigma;       // 高斯中心略低于最小残差
hypos(i,4) = probDesity(res_i, mu, sigma);     // 高斯概率密度
归一化 + 算累积概率 hypos(i,5);
```
> 💡 `mu` 取"最小残差 - 2σ"而非最小残差本身，是为了让概率分布**集中在残差最低的粒子附近**，加快收敛。累积概率列 `hypos(i,5)` 用于轮盘赌重采样。

### `genInitialParticles`（`locator.cpp:87`）—— 初始撒点
在搜索范围 `constraints`(PoseBox2D) 内**均匀随机**撒 `num`(默认 200) 个粒子。用系统时钟做随机种子。

### `genParticles`（`locator.cpp:102`）—— 收缩重采样
这是"越搜越细"的核心：
```cpp
int num = hypos.rows() * numShrinkRatio;        // 保留 85%
// 轮盘赌：按累积概率选粒子（概率高的更可能被选中，可重复）
for (i) { rand = 随机[0,1]; 找第一个累积概率≥rand的粒子j; 复制它; }
// 扰动逐轮缩小
offsetX *= offsetShrinkRatio;  // ×0.8
offsetY *= offsetShrinkRatio;
offsetTheta *= offsetShrinkRatio;
hypos += 随机扰动(offsetX, offsetY, offsetTheta);
// 钳制到搜索范围内
hypos.col(0).cwiseMax(xmin).cwiseMin(xmax);  ...
```
> 💡 **收缩搜索**：每轮保留概率高的粒子、在它们附近撒新点、且**扰动幅度逐轮 ×0.8 缩小**。相当于探照灯越照越聚焦到最有希望的区域。这比固定扰动的标准粒子滤波收敛快。

## 六、主循环 `locateRobot`（`locator.cpp:273`）

把上面串起来：
```cpp
LocateResult locateRobot(markers_r, constraints, numParticles=200, ...) {
    if (markers_r.size() < minMarkerCnt) return {fail, code=4};  // 角点太少，定不了

    genInitialParticles(numParticles);
    if (calcProbs(markers_r)) return {fail, code=5};             // 所有概率太低

    for (i = 0; i < maxIteration(20); i++) {
        res.residual = bestResidual / markers_r.size();          // 平均残差
        if (isConverged()) {                                     // 粒子都聚到一起了
            if (res.residual > residualTolerance) return {fail, code=2};  // 收敛了但残差大
            return {success, code=0, pose=bestPose};             // 成功！
        }
        if (genParticles()) return {fail, code=1};               // 粒子数归零
        if (calcProbs(markers_r)) return {fail, code=5};
    }
    return {fail, code=3};                                       // 20轮没收敛
}
```

`isConverged`（`:264`）：当所有粒子的 x/y/θ 跨度都小于 `convergeTolerance`(0.2) 才算收敛。

**返回码 `code` 的含义**（`types.h` 的 `LocateResult`）：
| code | 含义 |
|------|------|
| 0 | 成功 |
| 1 | 粒子数归零 |
| 2 | 收敛但残差超阈值（位姿不可信）|
| 3 | 20 轮未收敛 |
| 4 | 角点数 < `minMarkerCnt`（看到的太少）|
| 5 | 所有粒子概率太低 |

### `finalAdjust`（`locator.cpp:198`）—— 收敛后微调
收敛后可做小幅平移微调：把所有观测角点到最近真值角点的偏移取平均，加到位姿上（只调平移不调旋转）。

## 七、定位成功后干什么

`locateRobot` 只是算出位姿。**落地**由调用它的行为树节点完成（见 [6.2](./6.2-定位行为树节点.md)）：调 `brain->calibrateOdom(pose)` 把结果反推成 `odomToField`（见 [模块05 · 5.4](../05-大脑数据与坐标系/5.4-三套坐标系与变换.md)），置 `odom_calibrated=true`，更新 `lastSuccessfulLocalizeTime`。

---

## 小结

- `Locator` 用粒子滤波解"已知地图+观测，求位姿"，应对角点匹配的多峰歧义。
- 真值角点地图按场地尺寸算出，分 X/P/T/L 四类，同类间匹配。
- 残差 = 观测角点变换后到最近同类真值的距离（按距离加权），残差小→概率高。
- 收缩搜索：每轮保留高概率粒子、扰动逐轮 ×0.8 缩小，最多 20 轮，收敛且残差达标才成功。
