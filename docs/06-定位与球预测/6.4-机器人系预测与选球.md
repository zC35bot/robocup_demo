# 6.4 · 机器人系预测与选球流水线

[6.3](./6.3-球预测IMM.md) 的 IMM 预测器工作在**场地系**，前提是定位准。但定位会失效（刚摔倒、长时间没看到角点）。本篇讲它的"备胎"——机器人系预测器，两者怎么切换，以及一帧检测出多个"球"时怎么挑出真的那个。

---

## 一、为什么需要两套预测器

- **场地系 `BallImmPredictor`**（[6.3](./6.3-球预测IMM.md)）：能给 pred300、知道球往哪个球门去。**但前提是自定位准**——它的输入是球的场地坐标，定位错了场地坐标就全错。
- **机器人系 `RobotFramePredictor`**（`robot_frame_predictor.cpp`）：只给 pred100、**不依赖定位**——它的输入是球相对机器人的坐标。

> 💡 **核心权衡**：场地系预测更强大，但"地基"（定位）不稳时会塌。一旦刚摔倒爬起、或长时间没成功定位，场地坐标不可信，这时退回机器人系预测——"我不知道自己在场上哪，但我知道球在我前方 2 米往左滚"，至少还能追球。这是稳健性的关键设计：**强方案 + 弱但可靠的退路**。

---

## 二、机器人系预测器 `RobotFramePredictor`

`robot_frame_predictor.cpp`。它是 IMM 的简化版——**单模型卡尔曼**（只有滚动模型）：

| 方面 | 场地系 IMM | 机器人系 KF |
|------|-----------|------------|
| 模型数 | 2（静止+滚动）| 1（滚动，mu=0.3）|
| 加速度噪声 | 0.05 / 0.6 | 0.8（更大，因为机器人自身在动）|
| 遮挡衰减 | 有 | 有 |
| 输出 | pred100 + pred300 | 只 pred100 |
| 依赖定位 | 是 | 否 |

动力学和 IMM 的滚动模型一样（带摩擦的匀速 + 白噪声加速度），但加速度噪声更大（0.8）——因为机器人本体在走动，球相对机器人的运动更"乱"，要给滤波器更多容错。它不给 pred300，因为不知道场地朝向，长时预测没意义。

## 三、切换判据 `isLocalizationTrusted`（`brain.cpp:960`）

```cpp
bool Brain::isLocalizationTrusted() {
    if (!tree->getEntry<bool>("odom_calibrated")) return false;       // 从没定位成功过
    return msecsSince(data->lastSuccessfulLocalizeTime) < config->ballPredictor.localization_trust_msec(1500);
}
```
**定位可信 = 曾成功定位 且 距上次成功定位 < 1.5 秒**。超过 1.5 秒没定位成功，就认为场地坐标已经漂得不可信了。

## 四、每帧推进 `updateBallMemory`（`brain.cpp:866`）

`tick` 里调（见 [模块01 · 1.4](../01-启动与架构/1.4-main与多线程.md)）。它推进两个预测器并把检测喂给对应的那个：
```cpp
void Brain::updateBallMemory() {
    // 球太久没看到 → 忘记
    if (msecsSince(ball.timePoint)/1000 > ball_memory_timeout) {
        tree->setEntry("ball_location_known", false);
        tree->setEntry("ball_out", false);
    }
    updateRelativePos(data->ball);  // 刷新球的相对位置
    tree->setEntry("ball_range", data->ball.range);

    std::lock_guard<std::mutex> lock(predictorMutex_);   // 保护预测器共享状态
    if (ballPredictorForceReset_) {                       // 选球流水线判定"球瞬移"了
        imm_predictor_.reset();  robot_frame_predictor_.reset();
        ballPredictorForceReset_ = false;
    }
    double dt = msecsSince(lastBallPredictTime_)/1000.0;
    if (dt > 0 && dt < 1.0) {                              // 忽略异常 dt
        imm_predictor_.propagate(dt);  robot_frame_predictor_.propagate(dt);
    }
    lastBallPredictTime_ = now;

    if (data->ballDetected) {
        robot_frame_predictor_.add([ball.posToRobot], ball.range);   // 机器人系总是喂
        if (isLocalizationTrusted())
            imm_predictor_.add([ball.posToField], ball.range);       // 场地系仅定位可信时喂
        else
            imm_predictor_.handleOccluded();                          // 定位不可信，当作没看到
    } else {
        imm_predictor_.handleOccluded();  robot_frame_predictor_.handleOccluded();
    }
    updateBallPrediction();   // 把结果写进 BrainData/黑板
}
```
> 💡 注意 `predictorMutex_`：预测器在 tick 线程推进，但选球流水线在**检测回调线程**里读它的 `mahalanobis`（见下），所以共享状态要加锁。`ballPredictorForceReset_` 是检测线程通知 tick 线程"球瞬移了，重置滤波器"的标志位。

## 五、写出结果 `updateBallPrediction`（`brain.cpp:921`）

根据定位是否可信，把对应预测器的输出写进 `BrainData` 和行为树黑板：
```cpp
if (isLocalizationTrusted()) {
    auto p = imm_predictor_.getPrediction();
    data->filtered_ball_field = p.pos;

```

## 六、选球流水线 `detectProcessBalls`（`brain.cpp:2007`）

视觉一帧可能检测出多个"球"（含误检）。这个函数**不是简单选置信度最高的**，而是一条 6 步流水线（在检测回调线程里跑）：

```cpp
for (每个候选球 ballObj) {
    // 1. 置信度过滤
    if (ballObj.confidence < ball_confidence_threshold) continue;
    // 2. 场地边界过滤（球不可能在场外太远）
    if (|posToField.x| > L/2+2 || |posToField.y| > W/2+2) continue;
    // 3. 预测门控：算到 IMM 估计的马氏距离
    if (imm 已初始化 && 定位可信) {
        dM = imm_predictor_.mahalanobis([posToField], range);
        anomalous = (dM² > gateChi2(5.99));   // 卡方门控
    }
    // 5. 打分融合
    confNorm = conf/100;
    sizeWeight = clamp(sqrt(框面积)/30, 0.3, 1.2);   // 框太小可能误检，降权
    historyWeight = exp(-到上帧球的距离/2);            // 离上帧球越近越可信
    score = confNorm * sizeWeight * historyWeight * (anomalous ? 0.3 : 1.0);
    选 score 最高的为候选;
}
// 4. 跳变剔除 jump-reject
if (候选异常 || (连续帧 && 跳变距离>0.4m)) {
    consecutiveAnomalyCount++;
    if (count < 8) 丢弃当帧;                    // 保护滤波器不被瞬时误检带偏
    else { ballPredictorForceReset_ = true; }   // 连续8帧异常→认定球真瞬移，重置滤波器
}
// 6. 接受
if (有候选 && !rejected) {
    data->ballDetected = true;  data->ball = 候选;
    tree->setEntry("ball_location_known", true);
    更新 robotBallAngleToField;
}
```

逐步解读：
1. **置信度过滤**：低于阈值（默认 50）丢弃。
2. **场地边界过滤**：球不可能在场外 2 米以上（需要场地坐标，定位不准时此步可能误判）。
3. **预测门控**：用 [6.3](./6.3-球预测IMM.md) 的马氏距离判断候选是否偏离当前估计太远，标记 `anomalous`。
4. **跳变剔除 `jump-reject`**：和上帧球比，短时间跳变超 0.4m 且异常 → 丢当帧。**但连续 8 帧都异常** → 认为球真的"瞬移"了（被踢飞/重新摆球），接受并置 `ballPredictorForceReset_` 让滤波器重置。
5. **打分融合**：`score = 置信度 × 框大小权重 × 历史距离权重 ×(异常则×0.3)`。框太小降权、离上帧球近加权、异常重罚。
6. **接受最高分**为真球。

> 💡 这条流水线体现"**别盲信单帧检测**"：结合时间一致性（历史位置、预测门控、跳变检测）判断哪个才是真球。`jump-reject` 的"连续 8 帧才认瞬移"很巧妙——既能抵抗单帧误检（丢弃），又不会在球真被踢飞时永远卡住（8 帧后重置）。

球记忆超时（`ball_memory_timeout` 默认 2~3 秒没看到）会置 `ball_location_known=false`，触发行为树去"找球"（见 [模块07](../07-行为树与决策/index.md)）。`lose_ball`（超 2 秒没看到真球）是另一个更强的"球丢了"信号。

---

## 小结

- 两套预测器：场地系 IMM（强，需定位）+ 机器人系 KF（弱但可靠，不需定位），`isLocalizationTrusted`（定位成功且<1.5s）切换。
- `updateBallMemory` 每帧推进两个预测器、按定位是否可信喂检测、加锁保护。
- `detectProcessBalls` 是 6 步选球流水线：置信度→边界→马氏门控→跳变剔除→打分→择优，靠时间一致性抗误检。
- `jump-reject` 连续 8 帧异常才认"球瞬移"并重置滤波器。
