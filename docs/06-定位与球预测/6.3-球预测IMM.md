# 6.3 · 球预测 IMM（核心）

视觉每帧给出球的当前位置，但有噪声、会被遮挡，而决策需要"球**接下来**会到哪"才能提前跑位拦截。`BallImmPredictor`（`src/brain/src/pos_predictor.cpp`）用 **IMM（交互式多模型）卡尔曼滤波** 来平滑 + 预测。本篇把这套数学通俗讲透。

---

## 一、为什么需要它

视觉的球位置有三个毛病：① 有噪声（抖动）；② 会被遮挡（球被腿挡住几帧）；③ 是"现在"的位置，但机器人走过去要时间，到时候球早滚走了。

需要一个滤波器：**用历史观测平滑掉噪声、遮挡时合理外推、并预测未来位置**。

**关键洞察**：球只有两种运动状态——**静止**（停着）或**滚动**（被踢出去带摩擦减速）。但你事先不知道当前是哪种。IMM 的妙处就是**同时维护两个模型（静止/滚动），按观测自动判断当前更像哪种，并融合它们的估计**。

---

## 二、状态与两个模型

**状态向量** `x = [px, py, vx, vy]`：位置 + 速度（场地系）。
**观测** `z = [bx, by]`：检测到的球心位置。

两个模型（`reset()`，`pos_predictor.cpp:77`）：
```cpp
// 模型0 静止：强摩擦、极小加速度噪声
models_[0].mu = 6.0;       models_[0].sigmaA = 0.05;
// 模型1 滚动：弱摩擦(可配)、中等加速度噪声
models_[1].mu = friction_decay_hz(0.30);   models_[1].sigmaA = 0.6;
```
- `mu`：摩擦系数（速度衰减率）。静止模型 mu=6 极大，意味着"速度会被迅速抹掉"——它倾向于认为球不动。滚动模型 mu=0.3 较小，速度衰减慢——它倾向于认为球在匀速滚动。
- `sigmaA`：加速度过程噪声。滚动模型噪声大（球可能被再踢、撞墙改向），静止模型噪声小（停着的球不该乱动）。

**马尔可夫转移矩阵**（`pos_predictor.cpp:71`）描述两模型间切换概率：
```cpp
transMat_ << 0.95, 0.05,    // 静止→静止95%, 静止→滚动5%
             0.05, 0.95;    // 滚动→静止5%, 滚动→滚动95%
```

## 三、单模型的卡尔曼滤波（`Model::predict` / `update`）

### 预测 `Model::predict(dt, qScale)`（`:12`）
按"带摩擦的匀速运动"外推状态：
```cpp
double decay = 1.0 - mu*dt;  if (decay<0) decay=0;   // 速度衰减系数
// 状态转移矩阵 F：位置 += 速度·dt；速度 ×= decay
F(0,2)=dt; F(1,3)=dt; F(2,2)=decay; F(3,3)=decay;
x = F * x;
// 过程噪声 Q（白噪声加速度模型，按 dt 的幂次分布到位置/速度协方差）
Q(0,0)=Q(1,1)=dt⁴/4·s²;  Q(0,2)=...=dt³/2·s²;  Q(2,2)=Q(3,3)=dt²·s²;
P = F*P*Fᵀ + Q;
```
> 💡 `decay = 1 - mu·dt` 是摩擦的离散近似：速度每过 dt 乘以 decay。静止模型 decay 接近 0（速度迅速归零），滚动模型 decay 接近 1（速度缓慢衰减）。`P` 是协方差（不确定度），预测时增长（越外推越不确定）。

### 更新 `Model::update(z, R)`（`:45`）
用新观测修正状态（标准卡尔曼更新）：
```cpp
H = [1 0 0 0; 0 1 0 0];           // 只观测位置，不观测速度
y = z - H*x;                       // 创新（观测 - 预测）
S = H*P*Hᵀ + R;                    // 创新协方差
K = P*Hᵀ*S⁻¹;                      // 卡尔曼增益
x = x + K*y;                       // 修正状态
P = (I - K*H)*P;                   // 修正协方差
// 顺便算这次观测的高斯似然（这个模型有多"吃惊"）
like = exp(-0.5·md²) / sqrt(4π²·det(S));   // md² 是马氏距离平方
```
> 💡 **似然 `like`** 是 IMM 的关键：它衡量"在这个模型看来，这次观测有多正常"。如果球突然移动，静止模型会很"吃惊"（似然低），滚动模型则不吃惊（似然高）——这就让系统自动判断"球现在在滚"。

**测量噪声 R 随距离增大**（`measNoise`，`:99`）：
```cpp
sigma = 0.124 * detDist + 0.149;   R = sigma² · I;
```
远处的球检测误差大，所以 R 更大，卡尔曼会更不信任远处观测。这个线性公式是对相机精度的经验标定。

## 四、IMM 的三步舞

每帧两个动作：`propagate`（预测）和 `add`（有观测时更新）。

### propagate：混合 + 各自预测（`:105`）
```cpp
// ① 算预测的模式概率 c_j = Σ_i transMat(i,j)·prob_i
// ② 混合：每个模型 j 的初始状态 = 各模型按转移权重加权平均
for (j) {
    xMixed[j] = Σ_i w_ij · models_[i].x;   // w_ij = transMat(i,j)·prob_i / c_j
    PMixed[j] = Σ_i w_ij · (P_i + (x_i-xMixed)(x_i-xMixed)ᵀ);
}
// ③ 各模型用混合后的初值各自 predict
for (j) models_[j].predict(dt, occ_q_scale_);
```
> 💡 **"混合"是 IMM 区别于简单多模型的精髓**：每帧开始时，让两个模型"交流信息"——按马尔可夫转移把彼此的估计加权混进对方。这样模型切换（静止→滚动）时状态能平滑过渡，而不是各算各的互不相干。

### add：测量更新 + 模式概率更新（`:151`）
```cpp
if (!initialized_) {  // 第一次观测：用它初始化两个模型
    models_[j].x = [z, 0, 0];  P 设大初值; confidence_=1; initialized_=true; return;
}
// 每个模型各自更新 + 算似然
for (j) likelihood(j) = models_[j].update(z, R);
// 模式概率按 Bayes 更新：prob_j = likelihood_j · cbar_j / Σ
for (j) models_[j].prob = likelihood(j) * cbar_(j) / norm;
```
> 似然高的模型获得更高的模式概率。如果连续几帧球在动，滚动模型的 prob 会逐渐逼近 1——系统"认定"球在滚。

## 五、遮挡处理 `handleOccluded`（`:191`）

球没看到时（被挡/检测失败）不调 `add`，而调这个：
```cpp
frames_since_detection_++;
occ_q_scale_ *= occlusion_noise_growth(1.05);   // 过程噪声每帧膨胀 5%
confidence_ *= confidence_decay_rate(0.92);      // 置信度每帧衰减 8%
if (frames_since_detection_ > max_occluded_frames(50)) confidence_ = 0;  // 超50帧放弃
```
> 💡 遮挡时仍用 `propagate` 外推球的位置（球还在按惯性滚），但**膨胀不确定度、衰减置信度**——外推越久越不可信。超过 50 帧（约 1.66 秒 @30fps）就彻底放弃，置信度归零。这让"球短暂被挡"和"球真的不见了"得到不同对待。

## 六、融合输出 `getPrediction`（`:226`）

把两个模型按各自概率融合，输出决策需要的一切：
```cpp
fuse(x, P);    // x = Σ_j prob_j · models_[j].x（概率加权融合）
out.pos = [px, py];           // 平滑后的位置
out.vel = [vx, vy];           // 球速度
out.mode_prob = [pStat, pRoll];  // 静止/滚动概率
out.confidence = clamp(confidence_, 0, 1);
out.valid = initialized && frames_since_detection ≤ 50 && confidence > 1e-3;

// 未来位置外推（摩擦感知）
extrapolate(0.1, pred100);   // 100ms 后
extrapolate(0.3, pred300);   // 300ms 后
```

**外推公式**（`:255`）摩擦感知：
```cpp
if (pRoll >= pStat) {   // 滚动占优：用摩擦模型外推
    fx = vx*t - 0.5*mu*vx*t²;   // 位移随摩擦减少，但不反向
    if (fx*vx < 0) fx = 0;       // 钳制：摩擦项不能让位移反号
    out = p + f;
} else {                // 静止占优：球不动，预测=当前位置
    out = p;
}
```
> 💡 `p(t) = p + v·t - 0.5·μ·v·t²` 是带摩擦的位移公式：球边滚边减速。钳制 `if (fx*vx<0) fx=0` 防止 t 较大时摩擦项把位移算成负数（球不会因摩擦倒退）。静止占优时直接返回当前位置——停着的球预测它还在原地。

- **pred100**（100ms）：给追球用，补偿视觉+控制延迟（见 [模块07](../07-行为树与决策/index.md) Chase）。
- **pred300**（300ms）：给调整/踢球用（见 [模块07](../07-行为树与决策/index.md) Adjust）。

## 七、马氏距离门控 `mahalanobis`（`:211`）

供大脑判断"新检测是不是离谱"：
```cpp
fuse(x, P);   // 融合当前估计
y = z - H*x;  S = H*P*Hᵀ + R;
md = sqrt(yᵀ·S⁻¹·y);    // 马氏距离（考虑了不确定度的"标准化距离"）
```
大脑用它做**卡方门控**（阈值 5.99 = χ²₀.₉₅(2)）剔除离谱的检测——详见 [6.4](./6.4-机器人系预测与选球.md) 的选球流水线。
> 💡 马氏距离比欧氏距离好在哪？它用协方差 S 归一化——如果当前估计很不确定（P 大），门就宽容；如果很确定，门就严格。"离谱"是相对于当前不确定度而言的。

---

## 小结

- 球预测用 **IMM 双模型卡尔曼**：静止模型(强摩擦)和滚动模型(弱摩擦)同时跑。
- 每帧 `propagate`（混合 + 各自带摩擦预测）+ `add`（测量更新 + 按似然更新模式概率）。
- 似然让系统自动判断当前"球在动还是停"；遮挡时外推 + 膨胀噪声 + 衰减置信度，超 50 帧放弃。
- `getPrediction` 融合输出平滑位置 + 速度 + pred100/pred300（摩擦感知外推）+ 置信度。
- `mahalanobis` 做卡方门控剔除离谱检测。
