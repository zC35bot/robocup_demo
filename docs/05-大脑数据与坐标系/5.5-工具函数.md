# 5.5 · 工具函数（math / misc / print）

`src/brain/include/utils/` 下三个头文件提供了贯穿全项目的工具函数。它们都很短小，但在决策、几何、日志里被反复调用。本篇逐个过一遍。

---

## 一、`math.h` —— 数学与几何

### 角度
```cpp
deg2rad(deg) / rad2deg(rad)    // 角度 ↔ 弧度
toPInPI(theta)                  // 把角度规整到 [-π, π)
thetaToX(v)                     // 向量 v 相对 x 轴的角度
```
**`toPInPI` 是最常用的工具之一**：角度运算（如 `kickDir - robotAngle`）结果可能超出 [-π,π]，规整后才能正确比较大小。实现用 `fmod` 加偏移保证落在区间内：
```cpp
inline double toPInPI(double theta) {
    int n = static_cast<int>(fabs(theta/2/M_PI)) + 1;
    return fmod(theta + M_PI + 2*n*M_PI, 2*M_PI) - M_PI;
}
```
> 💡 为什么重要？比如判断"机器人对球方向"和"期望踢球方向"哪个更接近，若不规整，179° 和 -179° 会被当成差 358° 而非 2°，决策就错了。

### 基础运算
```cpp
mean(x,y) / mean(vector)       // 平均
cap(x, upper, lower)           // 钳制到 [lower, upper]——限速、限角到处用
norm(x,y) / norm(vector)       // L2 范数（距离）
sigmoid(x, shift, scale)       // S 形函数，平滑过渡
```
`cap` 在控制里无处不在：`vx = cap(vx, vx_limit, -vx_limit)`。`sigmoid` 用于让速度随某量平滑变化（如 [模块07](../07-行为树与决策/index.md) 追球时 `vx *= sigmoid(|vtheta|,1,3)`，转向大时减速）。

### 坐标变换（[5.4](./5.4-三套坐标系与变换.md) 已详讲）
```cpp
transform(p0, theta)           // 2D 点旋转
transCoord(...)                // SE(2) 位姿复合（最底层）
trans(x,y,theta, xf,yf,thetaf, dir)  // Eigen 矩阵版变换，forth/back
```

### 向量
```cpp
crossProduct(a,b) / innerProduct(a,b)   // 2D 叉积/点积
```

### 直线几何（自定位、场地线处理用，见 [模块06](../06-定位与球预测/index.md)）
```cpp
linesIntersect(l1,l2)          // 两线段是否相交（叉积符号法）
angleBetweenLines(l1,l2)       // 两线夹角
lineLength(l) / extendLine(l,len)        // 长度 / 两端延长
pointPerpDistToLine(p,l)       // 点到直线的带符号垂距
pointMinDistToLine(p,l)        // 点到线段的最小距离
boxDistToLine(box,l)           // 包围框到直线距离
isSameLine(l1,l2,...)          // 两段是否属于同一条线（角度+垂距+端点距）
probPartOfLine(l1,l2)          // l1 是 l2 一部分的概率
mergeLines(l1,l2)              // 合并两段为最长跨度
linearFit(x,y,calcR2)          // 最小二乘直线拟合 y=ax+b，可选算 R²
```
> 这些直线工具主要服务于**场地线识别与自定位**：把视觉给的零碎线段合并、判断属于哪条真实场地线（中线/边线/球门线）、用拟合做精化。`isSameLine` 综合了角度差、垂距、端点距三个条件，比单一判据稳健。

`angleBetweenLines` 用 `acos(点积/模长积)` 求两线夹角，但送进 `acos` 前先 `clamp` 到 `[-1, 1]`（`math.h:160`）：
```cpp
auto angle = acos(clamp((innerProduct(AB, CD)) / normProdut, -1.0, 1.0));
```
> 💡 数学上 `点积/模长积` 恒在 `[-1,1]`，但浮点舍入误差会让它偶尔算出 `1.0000001` 这类越界值；`acos` 一旦拿到 `|x|>1` 就返回 **NaN**，NaN 会顺着夹角一路污染 `isSameLine`、`msecsToCollide` 等所有下游判断且难以排查。先 `clamp` 到合法定义域，把这条数值隐患从根上堵死。（`clamp` 需要 `<algorithm>`，头部已一并补上。）

`linearFit` 用经典最小二乘公式：
```cpp
a = (n*Σxy - Σx*Σy) / (n*Σxx - Σx²);   b = (Σy - a*Σx) / n;
```

## 二、`misc.h` —— 杂项工具

```cpp
msecsSince(start_time)         // 距某时间点过去了多少毫秒——超时判断无处不在
gen_uuid()                     // 生成 UUID v4（RPC 调用追踪用，见模块08）
gen_timestamp_str()            // "YYYYMMDD_HHMMSS" 时间字符串
gen_timestamped_filename()     // 带时间戳的文件名（日志/数据集）
mkdir_if_not_exist(path)       // 不存在就创建目录
MergeYAML(base, override)      // 递归合并两个 YAML（后者覆盖前者）
```

**`msecsSince`** 是控制流里最高频的工具：判断"球多久没看到""定位多久没成功""指令冷却到了吗"全靠它。

**`gen_timestamped_filename`** 在拼接目录与文件名前，先挡掉空目录的情况（`misc.h:55`）：
```cpp
if (dir.empty()) return filename;      // 空目录：直接返回纯文件名
if (dir.back() == '/') return dir + filename;
return dir + "/" + filename;
```
> 💡 若传入的 `dir` 为空字符串，`dir.back()` 是对空 `std::string` 取末字符——**未定义行为**（可能崩溃或读到垃圾）。加一句 `if (dir.empty()) return filename;` 后，空目录就安全地退化为"只用文件名"，不再触碰 `back()`。

**`MergeYAML`** 实现了配置的**层叠覆盖**（递归合并）：
> 💡 这正是 [模块01 · 1.3](../01-启动与架构/1.3-launch文件逐行.md) 讲的配置层叠在 C++ 侧的实现之一——`loadConfig` 用它把 `vision_local.yaml` 叠加到 `vision.yaml` 上（见 [5.1](./5.1-Brain类与初始化.md)）。视觉模块也用它合并三个配置文件（见 [模块03](../03-视觉模块/index.md)）。

## 三、`print.h` —— 彩色日志与格式化

```cpp
RED_CODE / GREEN_CODE / ...    // ANSI 终端颜色码
format(fmt, ...)               // printf 风格的字符串格式化（返回 std::string）
vec2str(v)                     // 向量转 "[x, y, z]" 字符串
prettyPrint(title, ...)        // 带标题/颜色的控制台框
prtErr(msg) / prtWarn(msg) / prtDebug(msg)   // 醒目的错/警/调试块
```

**`format`** 是全项目最常用的字符串工具，几乎每条日志都用它拼接：
```cpp
log->debug("Chase", format("vx: %.2f vtheta: %.2f", vx, vtheta));
```
比 `std::stringstream` 简洁，比裸 `printf` 安全（返回 string）。它内部用一个固定 `char buf[1024]` 承接格式化结果，写入时改用**带长度上限**的 `vsnprintf`（`print.h:48`、`format2cstr` 在 `print.h:59`）：
```cpp
char buf[1024];
vsnprintf(buf, sizeof(buf), str, args);   // 旧代码是 vsprintf(buf, str, args)
```
> 💡 `vsprintf` 不知道 `buf` 有多大，一旦格式化结果超过 1024 字节就会**越界写栈、溢出缓冲区**（可能踩坏相邻变量甚至被利用）。`vsnprintf` 多传一个 `sizeof(buf)`，最多写 1023 字节再补 `\0`，超出部分安全截断。日志里偶尔出现的超长字符串（如打印整条轨迹、大 JSON）不再有把栈写爆的风险。

`prtErr/prtWarn/prtDebug` 给不同级别的输出加颜色和格式，让关键信息在一大片日志里一眼可见。

## 小结

- **`math.h`**：角度规整 `toPInPI`、钳制 `cap`、坐标变换 `transCoord`、一整套直线几何（自定位/场地线用）、最小二乘 `linearFit`；`angleBetweenLines` 对 `acos` 入参 `clamp` 到 `[-1,1]`，杜绝浮点越界产生 NaN。
- **`misc.h`**：超时判断 `msecsSince`、UUID、目录创建、配置层叠 `MergeYAML`；`gen_timestamped_filename` 加空目录守卫，避免对空串取 `back()` 的未定义行为。
- **`print.h`**：彩色日志、`format` 格式化、醒目错警块；`format`/`format2cstr` 改用 `vsnprintf`，格式化超 1024 字节时安全截断而非溢出栈缓冲区。

这些是后续所有模块代码里的"标准库"，看到它们直接对照本篇即可。至此模块 05 全部讲完。
