# 5.3 · BrainData 动态数据

`BrainData`（`brain_data.h` / `brain_data.cpp`）是大脑运行时一切**实时状态**的家。如果说 [`BrainConfig`](./5.2-BrainConfig静态配置.md) 是"出厂设置说明书"，`BrainData` 就是"此刻机器人眼里的世界"。本篇过一遍它的字段分类、线程安全设计和关键方法。

---

## 一、字段分类

### 比赛状态
```cpp
int score, oppoScore;          // 我方/对方比分
int secsRemaining;             // 剩余秒数
int penalty[HL_MAX_NUM_PLAYERS];     // 我方各球员罚则
int oppoPenalty[HL_MAX_NUM_PLAYERS]; // 对方各球员罚则
bool isKickingOff;             // 我方开球中
rclcpp::Time kickoffStartTime;
bool isFreekickKickingOff;     // 任意球开球中
int liveCount, oppoLiveCount;  // 我方/对方在场人数
string realGameSubState;       // 真实副状态（DIRECT_FREEKICK 等）
```
这些由 `gameControlCallback` 从裁判机消息更新（见 [模块04](../04-裁判机与通信/index.md)）。注意构造函数 `BrainData()` 把 `penalty[]` 初始化为 `SUBSTITUTE`（替补/不在场），意为"还没收到裁判机消息前，假设大家都不在场"。

### 自身位姿（三套坐标，详见 [5.4](./5.4-三套坐标系与变换.md)）
```cpp
Pose2D robotPoseToOdom;   // 机器人相对里程计原点（高频，会漂）
Pose2D odomToField;       // 里程计原点相对场地（定位校准）

double headPitch, headYaw;        // 头部角度

```

### 球
```cpp
GameObject ball;            // 当前球（位置/置信度/时间戳）
GameObject tmBall;          // 队友报告的球
bool ballDetected;
double robotBallAngleToField;  // 场地系下机器人指向球的方向
bool lose_ball;             // 球丢失（超2秒没看到）
// Phase1 球预测输出（详见模块06）
double filtered_ball_field[2];  // 滤波后场地系球位
double pred100_field[2], pred300_field[2];  // 100/300ms 预测（场地系）

```

## 二、线程安全：为什么列表要加锁

视野物体列表（机器人、门柱、角点、场地线、障碍）都用**互斥锁保护的 getter/setter**：

```cpp
inline vector<GameObject> getRobots() const {
    std::lock_guard<std::mutex> lock(_robotsMutex);
    return _robots;   // 返回拷贝
}
inline void setRobots(const vector<GameObject>& newVec) {
    std::lock_guard<std::mutex> lock(_robotsMutex);
    _robots = newVec;
}
```
每种列表配一把锁（`_robotsMutex`、`_goalpostsMutex`、`_markingsMutex`、`_fieldLinesMutex`、`_obstaclesMutex`），都是 `mutable`（const getter 里也能上锁）。

> 💡 **为什么必须加锁？** 回顾 [模块01 · 1.4](../01-启动与架构/1.4-main与多线程.md)：大脑是多线程的——**感知回调线程**（视觉检测回调）往这些列表里写，**100Hz 决策线程**（行为树）从里面读。两个线程同时访问同一个 `vector` 会数据竞争甚至崩溃。getter 返回**拷贝**而非引用，让调用方拿到一份当时的快照，用完即弃，避免持有引用期间被另一线程改掉。这是"读多写少、用拷贝换安全"的典型权衡。

注意：`ball`、`robotPoseToField` 等单值字段没加锁——它们是简单赋值，竞争窗口极小，项目选择容忍。球预测器的共享状态则用 `Brain::predictorMutex_` 单独保护（见 [模块06](../06-定位与球预测/index.md)）。

## 三、`GameObject`：一个场上物体的完整画像（`types.h`）

```cpp
struct GameObject {
    string label;         // "Ball"/"Opponent"/"Goalpost"/"LCross"...
    string color, name;   // 颜色 / 名字（如门柱 "OL"/"OR"）
    BoundingBox boundingBox;       // 图像像素框
    Point2D precisePixelPoint;
    double confidence;
    Point posToRobot;     // 相对机器人的三维位置
    Point posToField;     // 相对场地的三维位置
    double range;         // 距离
    double pitchToRobot, yawToRobot;  // 相对机器人的俯仰/偏航角
    rclcpp::Time timePoint;           // 何时检测到的
    int id; string info; double idConfidence;
};
```
> 一个 `GameObject` 同时携带**像素信息**(boundingBox)、**机器人系位置**(posToRobot/yawToRobot)、**场地系位置**(posToField)，以及**时间戳**。`timePoint` 很关键——决定这个观测多久算"过期"（球记忆超时、机器人列表清理都靠它）。

## 四、关键方法（`brain_data.cpp`）

### `getMarkingsByType(types)`（`:9`）
按类型筛选角点：传空集返回全部，否则只返回 label 在 types 里的。
```cpp
data->getMarkingsByType({"TCross", "XCross"});  // 只要 T 和 X 角点
```

### `getMarkersForLocator()`（`:24`）
把角点 `GameObject` 转成自定位用的 `FieldMarker`（label 字符串映射成单字符类型 L/T/X/P，带机器人系坐标和置信度）：
```cpp
if (label=="LCross") markerType='L';
else if (label=="TCross") markerType='T';
else if (label=="XCross") markerType='X';
else if (label=="PenaltyPoint") markerType='P';
res.push_back(FieldMarker{markerType, x, y, confidence});
```
这是喂给 [模块06](../06-定位与球预测/index.md) `Locator` 的输入。

### `robot2field(pose)` / `field2robot(pose)`（`:48`/`:59`）
机器人系 ↔ 场地系互转（基于当前 `robotPoseToField`），是决策代码最常用的坐标变换便捷封装。详细推导见 [5.4](./5.4-三套坐标系与变换.md)。

## 小结

- `BrainData` = 大脑此刻的世界快照：比赛状态、自身位姿、球、视野物体、踢球意图、队友、摔倒、各种时间戳。
- 视野物体列表用**互斥锁 + 返回拷贝**做线程安全（感知线程写、决策线程读）。
- `GameObject` 同时带像素/机器人系/场地系位置和时间戳。
- `getMarkersForLocator` 把角点喂给自定位，`robot2field/field2robot` 做坐标互转。
