# 5.2 · BrainConfig 静态配置

`BrainConfig`（`brain_config.h` / `brain_config.cpp`）封装所有**启动时确定、运行时只读**的配置。本篇过一遍它的配置分类、三种场地尺寸，以及它如何把场地真值地图算出来。

---

## 一、设计原则：静态只读

`brain_config.h:73` 的注释写明：
> 存放 `Brain` 需要的配置值，这些值在初始化时确定，决策过程中视为只读。运行时会变的值应放进 `BrainData`。

读取方式是一堆 `get_xxx()` 方法，内部就是 `brain->get_parameter("xxx").as_xxx()` 的封装。这样决策代码写 `config->get_vx_limit()` 而不是到处 `get_parameter("robot.vx_limit")`，既简洁又集中。配置值来自层叠的 yaml（见 [模块01 · 1.3](../01-启动与架构/1.3-launch文件逐行.md)）。

---

## 二、配置分类

### 比赛相关
```cpp
get_player_id() / get_team_id() / get_player_role()      // 球员号/队号/角色
get_field_type()        // "adult_size" | "kid_size" | "robo_league"
get_num_of_players()    // 每队人数
get_treat_person_as_robot()  // 调试：把检测到的"人"当对手机器人
get_game_control_ip()   // 裁判机 IP
```

### 机器人本体
```cpp
get_robot_name()        // 多机隔离用的名字（见 5.1）
get_robot_height()      // 0.8m (K1) / 1.12m (T1)——影响头部俯仰算球距
get_robot_odom_factor() // 里程计标度补偿 0.8(K1)/1.2(T1)（见 5.4）
get_vx_factor()         // x方向速度补偿（蟹步用，见模块08）
get_yaw_offset()        // 定位角度偏差补偿
get_vx_limit/vy_limit/vtheta_limit()   // 三方向限速
get_min_vx/min_vy/min_vtheta()         // 最小速度（太小机器人迈不动，给下限）
get_head_yaw_limit_left/right()  get_head_pitch_limit_up()  // 头部角度限位
```
> 💡 `robot_height` 和 `odom_factor` 是真机/T1 必改的两个值（根 README 特别强调）。前者因为算"球在视野里的俯仰角→球距"要用到相机高度；后者因为不同机器人步幅不同，里程计读数要乘一个标度才准。

### 策略
```cpp
get_ball_confidence_threshold()   // 球检测置信度门槛（默认50）
get_ball_memory_timeout()         // 球多久没看到就"忘记"（默认2~3秒）
get_tm_ball_dist_threshold()      // 采信队友球位的距离门槛
get_limit_near_ball_speed() / get_near_ball_speed_limit() / get_near_ball_range()  // 近球限速
get_abort_kick_when_ball_moved()  // 球动了是否中止踢球
get_ball_out_threshold()          // 球出界判定
get_enable_role_switch() / get_ball_control_cost_threshold()  // 协作（模块04）
get_enable_shoot()                // 是否启用射门决策
```

### 视觉踢球 RLVisionKick
```cpp
get_enable_auto_visual_kick()              // 总开关（仅K1+固件1.5.2+支持）
get_auto_visual_kick_enable_dist_min/max() // 触发距离范围
get_auto_visual_kick_enable_angle()        // 触发角度
```

### 定位器
```cpp
get_min_marker_count()   // 至少看到几个角点才定位
get_max_residual()       // 定位残差上限
```

### 避障（一大组）
```cpp
get_depth_sample_step() / get_obstacle_min_height() / get_grid_size()  // 深度采样/障碍栅格
get_max_x/max_y() / get_exclusion_x/y()                                // 栅格范围/自身排除区
get_ball_exclusion_radius/height()                                     // 球附近不算障碍
get_occupancy_threshold() / get_collision_threshold() / get_safe_distance()
get_avoid_during_kick/chase() / get_kick_ao_safe_dist() / get_chase_ao_safe_dist()
```
这些都喂给 [模块08](../08-机器人控制与底层/index.md) 的避障逻辑。

### Phase1 参数块（结构体形式）
不同于上面的 `get_xxx`，Phase1 参数直接以**结构体成员**暴露（在 `loadConfig` 里填充，见 [5.1](./5.1-Brain类与初始化.md)）：
```cpp
Phase1BallPredictorConfig   ballPredictor;  // 摩擦/遮挡/门控阈值 → 模块06
Phase1ChaseConfig           chase;          // predict_msec → 模块07
Phase1KickConfig            kick;           // 中止阈值 → 模块07
Phase1KickSelectorConfig    kickSelector;   // 精决策阈值 → 模块07
Phase1PowerShootConfig      powerShoot;     // 大力射门 → 模块07
Phase1CooperationConfig     cooperation;    // 协作滞回 → 模块04
Phase1HeadControllerConfig  headController; // 头控开关 → 模块08
Phase1TrainingLoggerConfig  trainingLogger; // 训练记录开关 → 模块08
```
每个结构体定义在 `brain_config.h:18` 起，带默认值。例如 `Phase1BallPredictorConfig` 含 `friction_decay_hz=0.30`、`gate_chi2_threshold=5.99` 等（详见 [模块06](../06-定位与球预测/index.md)）。

### 相机参数（深度相机内参，`brain_config.h` 末尾）
```cpp
int cameraImageWidth=1280, cameraImageHeight=720;
double depthCameraFx=643.898, depthCameraFy=643.216, depthCameraCx=649.038, depthCameraCy=357.21;
Eigen::Matrix4d camToHead;   // loadConfig 从 vision.yaml 读入（见 5.1）
```
大脑自己处理深度图（避障）要用这些内参，详见 [模块08](../08-机器人控制与底层/index.md)。

## 三、三种标准场地尺寸（`types.h`）

`FieldDimensions` 结构体描述场地的各种尺寸：
```cpp
struct FieldDimensions {
    double length;            // 场地长（球门到球门方向）
    double width;             // 场地宽
    double penaltyDist;       // 罚球点到底线距离
    double goalWidth;         // 球门宽
    double circleRadius;      // 中圈半径
    double penaltyAreaLength/Width;  // 罚球区
    double goalAreaLength/Width;     // 球门区
};
// 三种预设
const FieldDimensions FD_KIDSIZE   {9, 6, 1.5, 2.6, 0.75, 2, 5, 1, 3};
const FieldDimensions FD_ADULTSIZE {14, 9, 2.1, 2.6, 1.5, 3, 6, 1, 4};
const FieldDimensions FD_ROBOLEAGUE{22, 14, 3.6, 2.6, 2, 2.25, 6.9, 0.75, 3.9};
```
> 🏆 这些数字来自 RoboCup 各组别的官方规则。`field_type` 配置项决定用哪套。场地系坐标（见 [5.4](./5.4-三套坐标系与变换.md)）就建立在这些尺寸上——比如对方球门 x 坐标是 `length/2`。

## 四、`handle()`：选场地 + 算真值地图（`brain_config.cpp`）

`loadConfig()` 会调 `config->handle()`，它根据 `field_type` 选好 `fieldDimensions`，再调两个函数算出场地的**真值几何地图**：

### `calcMapLines()`
算出场地上所有真实线段（边线、中线、球门线、罚球区线、球门区线）的场地坐标，存进 `mapLines`。

### `calcMapMarkings()`
算出所有真实角点的位置和类型，存进 `mapMarkings`：
- `XCross`：中圈与中线的交点。
- `PenaltyPoint`：两个罚球点。
- `TCross`：罚球区/球门区/边线形成的 T 型交点。
- `LCross`：各区角和场地四角的 L 型交点。

> 💡 这些"真值地图"是**自定位的参照系**：[模块06](../06-定位与球预测/index.md) 的 `Locator` 把机器人看到的角点和这张真值地图去匹配，反推机器人在场上的位置。所以场地尺寸一变，地图就重算，定位才正确。

`print()` 方法把所有配置打印出来，启动时输出到日志便于核对。

---

## 小结

- `BrainConfig` = 启动时定、运行时只读的所有配置，用 `get_xxx()` 暴露。
- 分类：比赛/本体/策略/视觉踢球/定位/避障/Phase1 结构体/相机内参。
- `robot_height` 和 `odom_factor` 是真机/T1 必改项。
- `handle()` 按 `field_type` 选三种场地尺寸之一，并算出真值线/角点地图供自定位匹配。
